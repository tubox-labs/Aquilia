# aq i18n

Internationalization operations: initialize locale files, extract translation keys, check coverage, and compile catalogs.

## Usage

```bash
aq i18n <SUBCOMMAND> [OPTIONS]
```

## Subcommands

### aq i18n init

Initialize i18n in the current workspace. Creates `locales/` directory with starter translation files.

```bash
aq i18n init [OPTIONS]
```

| Option        | Alias | Description                                                       | Default   |
| ------------- | ----- | ----------------------------------------------------------------- | --------- |
| `--locales`   | `-l`  | Comma-separated locale list                                       | `en`      |
| `--directory` | `-d`  | Base directory for locale files                                   | `locales` |
| `--format`    | `-f`  | Translation file format: `json`, `yaml`, or `surp`                | `json`    |

Starter translations are provided for `en`, `fr`, `de`, `es`, and `ja`.

```bash
aq i18n init
aq i18n init --locales en,fr,de,ja
aq i18n init --directory translations --format yaml
aq i18n init --format surp
```

### aq i18n check

Validate i18n configuration and catalog structure.

```bash
aq i18n check
```

Reports:

- Default locale, available locales, fallback locale
- Catalog directories and their contents
- Missing key strategy
- Auto-detect and resolver order settings
- Per-locale file listing

### aq i18n inspect

Display current i18n configuration as JSON.

```bash
aq i18n inspect
```

### aq i18n extract

Extract translation keys from Python and Jinja2 template source files.

```bash
aq i18n extract [OPTIONS]
```

| Option          | Alias | Description                                         | Default                          |
| --------------- | ----- | --------------------------------------------------- | -------------------------------- |
| `--source-dirs` | `-s`  | Comma-separated source directories                  | `modules,controllers`            |
| `--output`      | `-o`  | Output file path                                    | `locales/en/messages.json`       |
| `--no-merge`    |       | Overwrite output instead of merging with existing   | `False` (merge enabled)          |

Scans for:

- `_("key")` / `_n("key", count)` in Python files
- `{{ _("key") }}` in Jinja2 templates
- `i18n.t("key")` / `i18n.tn("key", count)` in Python files
- `lazy_t("key")` / `lazy_tn("key", count)` in Python files

```bash
aq i18n extract
aq i18n extract --source-dirs modules,templates --output locales/en/messages.json
aq i18n extract --no-merge
```

### aq i18n coverage

Show translation coverage per locale compared to the default locale.

```bash
aq i18n coverage
```

Example output:

```
Translation Coverage Report
──────────────────────────────────────────────────
  Default locale (en): 42 keys

  ✅ en        100.0%  (42/42 keys)
  🔶 fr         85.7%  (36/42 keys)
  ⚠  de         71.4%  (30/42 keys)
  ❌ ja          0.0%  (0/42 keys)
```

### aq i18n compile

Compile JSON locale files to SURP binary format for faster startup.

```bash
aq i18n compile [OPTIONS]
```

| Option        | Description                                 | Default    |
| ------------- | ------------------------------------------- | ---------- |
| `--directory` | Source locales directory                    | `locales`  |
| `--output`    | Output directory for compiled `.surp` files | same as source |

```bash
aq i18n compile
aq i18n compile --directory locales
aq i18n compile --directory locales --output artifacts/locales
```

!!! note "SURP Requirement"
    Compiling to SURP requires the `surp` package: `pip install surp`

## Examples

```bash
# Initialize i18n with multiple locales
aq i18n init --locales en,fr,de,es,ja -f json

# Check configuration
aq i18n check

# Extract all translation keys
aq i18n extract -s modules,controllers,templates

# View coverage
aq i18n coverage

# Compile to SURP for production
aq i18n compile --output artifacts/locales
```
