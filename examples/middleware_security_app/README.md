# Middleware Security App

## Purpose

Demonstrates middleware policy composition for CORS, CSP, and rate limiting using real Aquilia middleware primitives.

## Architecture

- `workspace.py` wires typed security integrations.
- `SecurityPolicyService` models layered `RateLimitRule` behavior and `CSPPolicy`.
- Tests use a small fake request object that implements the methods the middleware extractors require.

## Setup

```bash
python -m pip install -e ".[dev]"
python -m pytest examples/middleware_security_app -q
```

## Run

```bash
cd examples/middleware_security_app
python -m uvicorn runtime:app --reload --port 8069
```

## Expected Behavior

Public GET requests are keyed by IP address, while API POST requests use the `X-API-Key` header for rate-limit identity.

## Common Pitfalls

- Middleware extractors depend on request shape; tests should provide the same methods and attributes used by the extractor.
- CSP strings should be generated by `CSPPolicy`, not hand-built.

## Extension Ideas

Add full ASGI middleware tests, CSRF token handling, proxy fix behavior, security headers, and session middleware.

## Related APIs

`RateLimitRule`, `RateLimitMiddleware`, `api_key_extractor`, `ip_key_extractor`, `CSPPolicy`, `CorsIntegration`, `RateLimitIntegration`.
